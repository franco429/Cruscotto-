🧪 TEST DI VERIFICA POST-INSTALLAZIONE
===============================================================================

TEST 1: Verifica Servizio Attivo
---------------------------------
1. Apri services.msc
2. Cerca "CruscottoLocalOpener"
3. Deve essere:
   - Stato: In esecuzione
   - Tipo avvio: Automatico
   - Accesso: Sistema locale o Utente corrente

TEST 2: Verifica Auto-Discovery
-------------------------------
1. Apri http://127.0.0.1:17654/config
2. Dovresti vedere tutti i percorsi Google Drive rilevati
3. Esempio output atteso:
   {
     "roots": [
       "G:\\Il mio Drive",
       "G:\\My Drive", 
       "G:\\",
       "C:\\Users\\[utente]\\Google Drive",
       ...altri percorsi rilevati...
     ]
   }

TEST 3: Test Apertura File
--------------------------
1. Vai nel Cruscotto SGI → Documenti
2. Clicca icona occhio su un documento
3. Il documento deve aprirsi istantaneamente
4. Controlla log: %APPDATA%\.local-opener\service.log

TEST 4: Test Riavvio PC
-----------------------
1. Riavvia il PC completamente
2. Dopo il riavvio, senza fare nulla:
   - Vai su http://127.0.0.1:17654
   - Il servizio deve essere già attivo
   - I percorsi devono essere già configurati

TEST 5: Test Resilienza
-----------------------
1. Apri Task Manager
2. Trova il processo CruscottoLocalOpener
3. Termina il processo
4. Entro 5-10 secondi deve riavviarsi automaticamente

===============================================================================
🔧 RISOLUZIONE PROBLEMI
===============================================================================

PROBLEMA: Percorsi non trovati
SOLUZIONE: 
- Attendi 5 minuti (retry automatico in corso)
- Esegui AGGIORNA-CODICE-SERVIZIO.bat
- Verifica che Google Drive Desktop sia avviato

PROBLEMA: Servizio non parte
SOLUZIONE:
- Verifica porta 17654 libera: netstat -ano | findstr :17654
- Controlla log errori: %APPDATA%\.local-opener\service-error.log
- Reinstalla: DISINSTALLA-LOCAL-OPENER.bat + installa-servizio-finale.bat

PROBLEMA: File non si apre
SOLUZIONE:
- Verifica percorso in http://127.0.0.1:17654/config
- Controlla che il file esista nel percorso Google Drive
- Verifica associazioni file Windows
